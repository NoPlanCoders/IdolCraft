# Gakumas Produce

Minecraft Forge 1.20.1 向けの、プロデュースカードとバフ管理を追加する MOD です。

## 概要

この MOD は、手帳アイテムを使ってカードを選択し、カード効果でスコア計算やバフ管理を行うプロデュース風の遊びを Minecraft に追加します。

## 主な機能

- プロデュース手帳アイテム（エンチャント風フォイル付き）
- カード67種：本家学マスの「センスプラン」スキルカードを網羅（白/銀/金/虹）。カード名・効果・解放Pレベルは本家wiki（スキルカード一覧）に準拠
  - 例：挑戦 / 決めポーズ / スタートダッシュ / 立ち位置チェック / バズワード / 国民的アイドル / 至高のエンタメ / 天賦の才 など
  - 効果メカニクス：ダメージ、集中/好調/絶好調、元気、追加使用、ドロー、消費体力減少、次カード2回発動、継続パラメータ（ターン毎ダメージ）
  - ※本Modに厳密な対応概念が無い一部効果（継続火力・SSR調達等）は近似・簡略化（各カードのコメントに注記）
- カード入手（コレクション制）：レア度別素材（白/銀/金/虹）を各レア度の材料からクラフト → 「スキルカード作業台」ブロックを右クリックしてGUIを開き、素材と同レア度かつPレベル解放済みの任意のカードを選んで素材1つを消費して習得 → 恒久コレクションに追加（カードアイテムを直接右クリックでも習得可）
- デッキ編成画面（Dキー）：**習得済み（入手済み）のカードのみ**から構成できるGUI（本家学マス寄せの白ベース＋すりガラス＋CMYアクセントUI、ホバー拡大・ボタンの呼吸アニメーションつき）
- 山札、手札、捨て札、除外カードの管理（通常カード／レッスン中1回カードで挙動が異なる）
- 集中、好調、絶好調などのバフ管理（好調・絶好調はリアルタイム秒数で管理、専用アイコン付きHUD表示）
- カードごとの使用条件（例：「集中3以上で使用可」）。満たさない場合は手札で減光表示され、選択自体がキャンセルされる
- 重複使用でスタックするパッシブ効果（演出計画・天真爛漫など、同じカードを複数枚使うほど効果が重複発動）
- 「スキルカード追加使用」ボーナス（ターンを終えず連続してカードを使用できる効果）
- プロデューサーランク（Pレベル）進行システム：経験値オーブの取得量に応じて上昇し（必要量はレベルごとに逓増）、Pレベルに応じて使用可能カードが増える。Minecraft本来の経験値レベルは消費せず温存する
- プロデューサーランク画面（Pキー）：学マス風の白ベース＋すりガラス独自UIで、現在のPレベル・経験値バー・Pレベルで解放されるカード一覧（解放/未解放）を表示。解放Pレベルは本家学マスのスキルカード一覧に準拠
- HUD 表示（手札・バフアイコン）、手札・デッキ編成の両方でホバー時にカード詳細ポップアップを表示
- キーバインドによるカード操作（R: デッキリセット / G: カードスキップ / D: デッキ編成画面を開く / P: プロデューサーランク画面を開く）
- 睡眠時のデッキリセット
- Forge Capability によるプレイヤーデータ保存

## 対応環境

- Minecraft 1.20.1
- Minecraft Forge 47.3.0 以上
- Java 17

## 開発環境の起動

```powershell
.\gradlew.bat runClient
```

## ビルド

```powershell
.\gradlew.bat build
```

ビルドした MOD jar は `build/libs/` に出力されます。

## リポジトリ構成

```text
gradle/                 Gradle wrapper
src/main/java/          MOD の Java ソースコード
src/main/resources/     assets、lang、models、textures、mods.toml
build.gradle            ForgeGradle のビルド設定
gradle.properties       プロジェクト設定
settings.gradle         Gradle プロジェクト設定
```

## Git 管理しないもの

以下はローカルで生成されるため、Git 管理しません。

- `.gradle/`
- `build/`
- `run/`
- ローカルのテストワールド
- ビルド済み jar

テストワールドを残したい場合は、リポジトリ外にバックアップしてから作業してください。必要になったら `run/saves/` に戻して `runClient` を起動します。
